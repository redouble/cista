# 🔒 密匣 (Cista) — 零知识端到端加密文件共享系统

> **匣中剑气，唯你可启。**

密匣是一个开源自托管 Web 应用，提供零知识端到端加密文件共享。文件在**上传前**就在您的浏览器中完成加密——服务器永远无法查看您的明文数据。

> **无需终端，无需 OpenSSL，无需技术知识。会用网站就会用密匣。**

## ✨ 功能特性

### 🔑 一键密钥生成

使用浏览器原生 Web Crypto API 直接生成 RSA-4096 或 RSA-2048 密钥对。**无需命令行，无需技术知识。** 您的私钥永不离开您的设备。

### 🔒 端到端加密

- **AES-256-GCM** 使用独立的随机密钥对每个文件进行加密
- **RSA-4096 或 RSA-2048** 使用接收人的公钥加密文件加密密钥
- 文件在**上传前**即在浏览器中加密——服务器仅存储密文
- 解密在**下载后**的浏览器中完成

### 📤 通过分享码安全分享

- 生成 6-8 位字母数字分享码（大写字母+数字，排除易混淆字符）
- **两种分享模式：**
  1. **对已注册用户** — 使用接收人公钥加密，安全性最高
  2. **对未注册用户（一次性分享）** — 生成临时密钥对，**无需登录即可下载**
- 首页和顶栏都有「兑换分享码」入口，任何人可直接访问
- 可配置有效期限和下载次数
- 随时撤销分享码

### 👥 完整用户管理

基于邮箱的注册和登录、密码重置流程、账号删除（同步清理所有数据）、管理后台。

### 🌐 多语言支持

支持中文和英文两种语言，易于扩展更多语言。

### 📦 技术栈

| 层         | 技术                                               |
| ---------- | -------------------------------------------------- |
| 前端       | TypeScript, HTMX, Alpine.js, Tailwind CSS          |
| 后端       | TypeScript (Node.js), Express.js                   |
| 数据库     | SQLite (开发) / PostgreSQL (生产) — 通过抽象层     |
| 加密       | Web Crypto API (浏览器端), bcrypt (服务端)         |
| 文件存储   | 本地文件系统 (开发) / S3 兼容 (生产)               |
| 认证       | JWT (HTTP-only Cookie)                             |

## 🚀 快速启动

### 前提条件

- Node.js 18+
- npm

### 一键安装

```bash
git clone https://github.com/your-org/cista.git
cd cista
bash scripts/setup.sh
```

### 手动安装

```bash
# 安装依赖
cd server && npm install && cd ..
cd client && npm install && cd ..

# 创建数据目录
mkdir -p server/data

# 运行数据库迁移
cd server && npx tsx src/db/migrate.ts && cd ..

# 创建管理员用户（默认: admin@cista.local / Admin123!）
cd server && npx tsx src/db/seed.ts && cd ..

# 启动服务器
cd server && npx tsx src/index.ts
```

服务器运行在 **http://localhost:3000**。

### 环境变量

| 变量               | 默认值               | 说明                 |
| ------------------ | -------------------- | -------------------- |
| `PORT`             | `3000`               | 服务端口             |
| `HOST`             | `0.0.0.0`            | 服务主机             |
| `NODE_ENV`         | `development`        | 运行环境             |
| `DATABASE_URL`     | `./data/cista.db`    | SQLite 数据库路径    |
| `JWT_SECRET`       | 自动生成             | JWT 签名密钥         |
| `STORAGE_TYPE`     | `local`              | 文件存储后端         |
| `MAX_FILE_SIZE`    | `1073741824` (1GB)   | 最大上传大小         |
| `ADMIN_EMAIL`      | `admin@cista.local`  | 初始管理员邮箱       |
| `ADMIN_PASSWORD`   | `Admin123!`          | 初始管理员密码       |

## 🔐 安全设计

### 零知识架构

```
┌─────────────────────┐          ┌──────────────────────┐
│     发送者浏览器    │          │        服务器        │
│                     │          │                      │
│  1. 生成文件密钥    │ ──────►  │  仅存储：            │
│  2. 使用 AES-256-   │  上传    │  • 加密后的文件      │
│     GCM 加密文件    │          │  • 加密后的文件密钥  │
│  3. 使用接收人公钥  │          │  • 公钥              │
│     加密文件密钥    │          │  • 密码哈希          │
│  4. 清除明文文件和  │          │                      │
│     文件密钥        │          │  无法解密：          │
│                     │          │  • 文件 ✗            │
│                     │          │  • 文件密钥 ✗        │
└─────────────────────┘          │  • 私钥 ✗            │
         │                       └──────────────────────┘
         │    分享码                      │
         ▼                                ▼
┌─────────────────────┐          ┌──────────────────────┐
│     接收者浏览器    │◄─────────│  (或分享码链接)      │
│                     │  下载    │                      │
│  1. 下载加密文件    │          │                      │
│  2. 使用私钥解密    │          │                      │
│     文件密钥        │          │                      │
│  3. 使用文件密钥    │          │                      │
│     解密文件        │          │                      │
└─────────────────────┘          └──────────────────────┘
```

### 加密算法

| 用途           | 算法                             |
| -------------- | -------------------------------- |
| 文件加密       | AES-256-GCM（每文件随机 IV）     |
| 密钥加密       | RSA-OAEP (SHA-256)                |
| 密钥生成       | RSA-4096 / RSA-2048                |
| 密钥派生       | PBKDF2（600,000 次迭代）         |
| 密码哈希       | bcrypt（cost 10）                |

### 安全措施

- 所有私钥**仅在浏览器中**生成和存储（IndexedDB）
- 私钥使用用户密码派生的密钥进行加密存储
- 服务器零知识：不存储明文文件、私钥或密码
- JWT 令牌使用 HTTP-only Cookie（JavaScript 不可访问）
- 认证接口限流保护
- 密码强度要求
- 完全可审计——全部源代码开放
